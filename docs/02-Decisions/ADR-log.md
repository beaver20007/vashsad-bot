## ADR-001: aiogram 3.x как основной фреймворк

**Дата:** 2026-05
**Статус:** Принято

**Контекст:** Нужен Python-фреймворк для Telegram-бота с поддержкой FSM.

**Решение:** aiogram 3.x
- Нативная async/await поддержка
- Встроенный FSM (Finite State Machine) для диалогов
- Роутеры для разделения хендлеров
- Активное сообщество, хорошая документация

**Альтернативы:** python-telegram-bot (синхронный), telebot (устаревший)

## ADR-002: Claude API (Anthropic) для AI

**Дата:** 2026-05
**Статус:** Принято

**Контекст:** Нужен AI для чата по садоводству и анализа фото растений.

**Решение:** Claude claude-sonnet-4-20250514
- Vision поддержка (фото-диагностика растений)
- Качественные ответы на русском языке
- Адекватная стоимость (~$3/млн токенов)
- Уже используется в ДомБоте (есть опыт интеграции)

**Примечание:** Звонки через HTTP (aiohttp), не через SDK — меньше зависимостей.

## ADR-003: SQLite → PostgreSQL поэтапно

**Дата:** 2026-05
**Статус:** Принято

**Контекст:** Нужна база данных для хранения пользователей и заявок.

**Решение:** SQLite для MVP, PostgreSQL для production.
- SQLite: нет отдельного сервиса, проще развёртывание
- Переход на PostgreSQL когда MAU > 1000 или нужна многопоточность
- Storage layer (services/storage.py) абстрагирован — замена прозрачная

## ADR-004: In-memory FSM Storage для MVP

**Дата:** 2026-05
**Статус:** Принято (временно)

**Контекст:** FSM требует хранения состояний диалогов.

**Решение:** MemoryStorage (aiogram) для MVP.
- Нет внешних зависимостей
- Данные теряются при рестарте — допустимо для MVP
- Переход на RedisStorage когда Redis будет в стеке

## ADR-005: Лимиты Free-тира

**Дата:** 2026-05
**Статус:** Принято

**Контекст:** Нужно управлять расходами на Claude API.

**Решение:**
- AI-чат: 10 сообщений/мес
- Фото-диагностика: 3 фото/мес
- Подбор растений: 3 запроса/мес

**Обоснование:**
- При 1000 пользователях и среднем потреблении ~500 токенов/сообщение
- Расходы ~$15/мес на Claude API (приемлемо)
- Достаточно для демонстрации ценности продукта

## ADR-006: Природный стиль как ниша позиционирования

**Дата:** 2026-05
**Статус:** Принято

**Контекст:** Нужно отличаться от конкурентов.

**Решение:** Специализация на природном стиле садов.
- Конкретная ниша с идеологией — проще привлекать «своих» клиентов
- Меньше конкуренции чем «универсальный дизайнер»
- Системный промпт AI настроен под природный стиль
- Все тексты бота отражают эту специализацию

## ADR-007: Уведомления дизайнеру через Telegram

**Дата:** 2026-05
**Статус:** Принято

**Контекст:** Дизайнер должен получать уведомления о новых заявках.

**Решение:** bot.send_message(DESIGNER_TELEGRAM_ID, ...)
- Мгновенное уведомление в Telegram
- Не нужен отдельный email/CRM на старте
- Легко расширить (добавить кнопку «принять заказ») позже
- DESIGNER_TELEGRAM_ID в .env — безопасно
